Python is a good case study of what happens when programmers use the wrong tool.

Python is massively popular and widely used in many areas of programming:

"Python is used by the novice programmer as well as by the highly skilled professional developer. It is being used in academia, at web companies, in large corporations and financial institutions. Python is known for its general-purpose nature that makes it applicable in almost each domain of software development. Python as a whole can be used in any sphere of development."
PYTHON CURRENT TREND APPLICATIONS- AN OVERVIEW

And this is not for lack of good reason. Python is a versitile language that is simple to develop and deploy. Many people consider it an ideal language due to fast development times and a variety of useful libraries. However, this does not mean python is always the right choice for a project.

"When you have a hammer, everything starts to look like a nail."

Python is a slow language compared to lower level languages like C++ and even Java. Pythonâ€™s container based typing system is memory intensive. Python is not well suited for parallel processing and struggles at large scale data processing. This makes Python an all around poor choice for lower-level systems based tasks and a questionable option for large scale deployment. Perhaps the real reason so many older programmers and traditionalists do not like Python is because the language does so much of the hard work for you. While this is convenient, it means where other languages would force the user to content with some in depth coding principals python simply handles them behind the scenes. This makes aforementioned old school programmers upset because they most definitely had to contend with these difficult topics and feel that everyone else should as well.
