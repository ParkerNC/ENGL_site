EACH PROGRAMMING LANGUAGE HAS STRENGTHS AND WEAKNESSES

A common fallacy in programming is to pick a langauge and use it for everything. Think of each programming language as a tool, where each tool is good for a set of things and not so good for other things. A good programmer would pick the right tool for the job!

First, there are different types of programming. Each of these genres of types of programming is called a paradigm.

Different paradigms encapsulate different ways of thinking about a problem, which naturally have different strengths and weaknesses. Imperative programming represents the solution to a problem as a list of steps to solve it, while declarative programming represents a solution as a composition of smaller sub-problems.

On top of paradigms, each langauge has strengths and weaknesses.

Python is good for some applications, but bad for others. As you'll note on the chart, it's interpreted and dynamically typed, which is good for prototyping. Its slow speed and memory inefficiency make it a bad candidate for large projects.

Choosing the right language for the job is crucial to good design. As above, Python is not a good candidate for large-scale computationally-intensive programs. For cases like this, a language like C++.

Your favorite language is not the best! Programming languages evolved for different reasons and to solve different problems.

Timeline of programming languages.

1940's

  • The world’s first real programming language, called “Plankalkul'', was developed. It allowed for the creation of procedures.
  • Assembly Language was used to simplify the language of machine code.

1950's

  • The Algorithmic Language (ALGOL) was created, and served as the precursor for future languages such as C, C++, and Java. Another language, “LISP”, was created, and is still used by some companies today.
  • Common Business Oriented Language (COBOL) was created to operate all types of machinery used for finance, administration, and government.

1960's

  • Beginner’s All Purpose Symbolic Instruction Code (BASIC) was developed in order to enable students who did not possess strong mathematical or computer skills. BASIC was later implemented by Bill Gates and Paul Allen when they created Microsoft.

1970's

  • Pascal was developed. Designed to be easy to use and help in learning computers, this program was implemented by Apple during their early days.
  • C was developed, and was considered to be the first high-level programming language. Many popular languages used today are derivative of C, including Java, JavaScript, and Python.
  • SQL was developed to allow for better interactions with information stored in a database.

1980's

  • Objective-C is developed and used as the main language in writing Apple’s operating systems.
  • C++ is created as a modified version of C. It added features such as classes, functions, and templates. It is still considered one of the best programming languages ever created today.
  • Perl is written as a general purpose language, used for such things as CGI, databases, network and graphic programming.

1990's

  • Haskell is created as a heavily mathematical based programming language. Today it is used in applications that involve lots of calculations, numbers crunching, and records.
  • Python is a very popular language developed for building software, websites, and conducting data analysis. Its general purpose nature makes it popular for creating a wide variety of programs.
  • Java, one of the world's most popular programming languages, is developed. It is now used for constructing a multitude of applications on devices such as laptops and cell phones.
  • JavaScript is developed as a language for adding complex features to a website.

2000's

  • C#, similar to C,C++, and Java, is a general purpose programming language that allows developers to construct many different applications, and is used heavily by programmers in the making of games.

And this is not for lack of good reason. Python is a versitile language that is simple to develop and deploy. Many people consider it an ideal language due to fast development times and a variety of useful libraries. However, this does not mean python is always the right choice for a project.

"When you have a hammer, everything starts to look like a nail."

Python is a slow language compared to lower level languages like C++ and even Java. Python’s container based typing system is memory intensive. Python is not well suited for parallel processing and struggles at large scale data processing. This makes Python an all around poor choice for lower-level systems based tasks and a questionable option for large scale deployment. Perhaps the real reason so many older programmers and traditionalists do not like Python is because the language does so much of the hard work for you. While this is convenient, it means where other languages would force the user to contend with some in depth coding principals python simply handles them behind the scenes. This makes aforementioned old school programmers upset because they most definitely had to contend with these difficult topics and feel that everyone else should as well.

This is not to say this viewpoint is entirely wrong. Topics like memory management, typing, and parallelization are important in Computer Programming and should not be ignored.
